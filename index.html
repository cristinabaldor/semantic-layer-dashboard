Data Pipeline Glossary

Model
dbt · SQL transformation

A dbt model is a SELECT statement saved as a .sql file. Models transform raw source data into clean, business-ready tables — dims, facts, and bridges — that power downstream metrics and dashboards.

Dimension
dim_* · descriptive attributes

Dimension models describe the “who, what, where” of your data — people, places, courses, and dates. They’re the reference tables that fact models join to for context and filtering.

Fact Table
fct_* · events and transactions

Fact models capture the “what happened” — attendance events, assessment scores, staff observations. Each row is a measurable event, joined to dimensions for context.

Semantic Layer
Cube

The semantic layer sits between the data warehouse and BI tools. It defines business logic — joins, filters, and metric formulas — in one place so every tool sees consistent numbers.

Metric
measure · computed value

A metric is a computed value defined in the semantic layer: attendance rate, assessment proficiency %, staff retention. Topline metrics are the headline numbers tracked at the org level.

Next Gen Data Tools
business intelligence · LLMs · apps

Using a fully-defined set of dimensions, facts, and metrics, we create tools that users interact with directly. This is the final output — blocked until all upstream models and measures in its domain are complete.
